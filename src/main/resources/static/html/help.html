URL shortener help

This is prototype of URL shortener web service. To run it just enter command gradlew bootRun in project root or gradlew build and then java -jar build/libs/urlshort-0.0.1.jar.

Application will bind to address 127.0.0.1:8080 and have prefix /short/ by default but you can change this in src/main/resources/application.properties file. This file also includes admin's login and password (they are 'admin' and 'admin' by default) which can be used to access any account's statistic. Later in this document default settings will be assumed.

First you must register an account by performing a POST request to URL http://127.0.0.1:8080/short/account with chosen account's id in JSON body {"AccountId":"{AccountId"}. On success you will receive password for that account in response body: {"success":true,"description":"Your account is opened","password":"{password}"}.

Send a POST request to URL http://127.0.0.1:8080/short/register with authorization header which includes your account's id and password. JSON body of this request must include url and redirect type: {"url":"{url}","redirectType":{redirectType}}. URL can be anything and redirect type must be 301 or 302. You can omit redirect type altogether, it will be 302 by default then. In response JSON body there will be short url {"shortUrl":"{shortUrl"}. You can use that to redirect to registered url.

To view account's statistic go to http://127.0.0.1:8080/short/statistic/{accountId}. You must authorize either using accessed account's credentials or by using admin's login and password. Response JSON body will contain long URLs and counts of redirects to them like this: {"{url1}":{count1},"{url2}":{count2},...}.

All data in this prototype is stored in memory so it will be lost on application restart.
